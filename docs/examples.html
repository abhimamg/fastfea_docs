Examples

In this section you will see different configurations of yml file that can be used by the user. Copy the the suitable section and save it as input.yml file.

Single Case
 1MODULE: ODB_READER
 2ABAQUS_PATH: abaqus
 3---
 4CASE_NAME:  ABC
 5ODB_PATH:   uhbvalid_2020.odb
 6STEP_NAME:  LOAD
 7FRAME:      LAST
 8SET_TYPE:   ELEMENT
 9SET_NAME:   PART-1-1.PIPELINE
10POSITION:   INTEGRATION POINT
11OUTPUT:     [ESF1, SM]

You can add cases by repeating information under the CASES keyword. Here you want to extract field outputs for two steps.

Multiple Cases
 1MODULE: ODB_READER
 2ABAQUS_PATH: abaqus
 3---
 4CASES:
 5  - CASE_NAME:  Design Step
 6    ODB_PATH:   uhbvalid_2020.odb
 7    FRAME:      LAST
 8    STEP_NAME:  Step-16
 9    SET_TYPE:   ELEMENT
10    SET_NAME:   PART-1-1.CP_B_ELS
11    POSITION:   INTEGRATION POINT
12    OUTPUT:     [ESF1, SM, EE, SK]
13
14  - CASE_NAME:  Trawl Step
15    ODB_PATH:   uhbvalid_2020.odb
16    FRAME:      LAST
17    STEP_NAME:  Step-18
18    SET_TYPE:   ELEMENT
19    SET_NAME:   PART-1-1.CP_B_ELS
20    POSITION:   INTEGRATION POINT
21    OUTPUT:     [ESF1, SM, EE, SK]

If you notice, a few information has been included twice in both the cases. A cleaner way to write the same file would be:

Multiple Cases (Cleaner Way)
 1MODULE: ODB_READER
 2ABAQUS_PATH: abaqus
 3---
 4ODB_PATH:   uhbvalid_2020.odb
 5FRAME:      LAST
 6STEP_NAME:  Step-16
 7SET_TYPE:   ELEMENT
 8SET_NAME:   PART-1-1.CP_B_ELS
 9POSITION:   INTEGRATION POINT
10OUTPUT:     [ESF1, SM, EE, SK]
11
12CASES:
13  - CASE_NAME:  Design Step
14    STEP_NAME:  Step-16
15
16  - CASE_NAME:  Trawl Step
17    STEP_NAME:  Step-18
